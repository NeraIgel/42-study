[gnl]
-. 매개변수 검사
-. buf 동적할당 (외부에서 정의하는 매크로상수의 크기를 알 수 없기 때문에 엄청 큰 값이라면 정적배열로 선언 시 스택 오버플로우 위험이 있음)
-. while loop
   루프 탈출 조건은 아래 3가지 중 하나이다.
	a. line 발견
	b. line 발견하지 못하고 EOF 도달
	c. 파일읽기 오류 또는 동적할당 오류 발생
-. 후처리 작업
   위의 루프에서의 탈출 조건 3가지 경우에 대응하는 후처리 작업을 수행한다.
	a. 정적변수 문자열에서 line 잘라내고 재할당
	b. 정적변수 문자열 데이터를 line에 복사시키고 정적변수는 빈 문자열 처리
	c. 파일 읽기 오류 : 읽어들인 부분까지는 처리 가능하므로 후처리 동작 수행함
	   동적 할당 오류 : 정적변수 문자열 처리를 할 수 없으므로 후처리 동작 수행하지 않음
-. 리소스 정리 작업
   함수 종료 시 반드시 메모리 해제를 하기 위해서 루프에서 오류 발생 시 break 문을 사용하였다.
	a. buf 할당 해제
	b. EOF 도달 하였다면 정적변수까지 할당 해제
